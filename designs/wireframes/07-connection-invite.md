# 07 — Connection Invite (發起連結)

**對應:** M6 雙向連結機制 · **元件:** Modal over current page

**核心哲學:** 連結不是追蹤。每一次連結邀請都必須說清楚「為什麼」。這一句話就是關係的起點。

```
   │                                                              │

   │     ┌────────────────────────────────────────┐               │

   │     ⓘ 今日還可發送 2 次連結邀請 (每日上限 3 次)              │

   │     [ 取消 ]    [ 送出連結邀請 ] primary                     │
   │                                                              │
   └──────────────────────────────────────────────────────────────┘
```

### 規格
- 留言 **必填**,14–140 字
- 最長 140 字 (刻意類似一則短訊息,避免寫成長信)
- 每日發送上限:**3 次** (防刷、維持品質)
- 引用卡片選填,但 UI 鼓勵選(因為那才是共振的載體)
- 送出後按鈕變 disabled,toast:「已送出。對方同意後會通知你。」

### 錯誤態
| 場景 | 訊息 |
|---|---|
| 今日配額已滿 | 「你今天已經送出 3 次邀請,我們希望每次連結都是認真的。明天再試。」 |
| 對方已屏蔽你 | 模糊化,只顯示「無法送出」(不洩漏屏蔽狀態) |
| 對方帳號已刪除 | 「這位使用者已不在共振上。」 |

## 2. 接收方視角 — 🔔 通知列

```
┌────────────────────────────────────────────────────────────────┐
│  🔗 連結邀請                                                   │
│                                                                │
│  ┌────────┐                                                    │
│  │   ○    │  小柚 · 台灣 想和你連結                            │
│  └────────┘                                                    │
│                                                                │
│  「你寫的那碗湯,讓我想起我離家那年的第一頓早餐。我想告訴      │
│   你這個故事。」                                               │
│                                                                │
│  引用了你的卡片:《那些深夜的一碗味噌湯》                      │
│                                                                │
│  [ 看看小柚是誰 ] ghost   [ 之後再說 ] ghost                   │
│  [ 接受連結 ] primary                                          │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### 動作
- **接受** → 雙方進入「已連結」狀態,各自看到對方的完整卡片 + 私訊通道開啟。Toast:「你和 小柚 已連結。開始一段對話吧。」
- **看看小柚是誰** → 跳至 [06 profile-other 未連結態](06-profile-other.md),可先讀公開卡再決定
- **之後再說** → 通知標為已讀,但邀請仍保留 7 天
- **不感興趣** (在 ⋯ menu 內) → 靜默忽略,發起方不會被告知被拒

### 拒絕時的保護設計
發起方看不到「被拒絕」,只會看到:
- 對方 7 天內未回應 → 邀請自動過期,只通知發起方「這次邀請過期了,有時就是時機不對」
- 永遠不顯示「被拒絕」訊息 — 降低挫敗與報復動機

## 私訊 (Phase 2 初版)

超出 MVP 核心範圍,但連結後基本通道:

- 一對一純文字 + 引用卡片
- 不顯示已讀 (避免壓力)
- 無貼圖、語音、視訊
- 可隨時靜音 / 解除連結
